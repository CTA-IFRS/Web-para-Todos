Landmark Roles

Esse tipo de role é utilizado para caracterizar pontos de referência numa página, marcos de navegação. Vejamos o seguinte exemplo:

        <!-- Cabeçalho da Página -->
<header role="banner">
<h1>Título da Página</h1>

<!-- Navegação -->
<ul role="navigation">
...
</ul>
</header>

<!-- Conteúdo Principal -->
<main role="main">

<!-- Postagem -->
<section>

<!-- Cabeçalho da Postagem -->
<header>
<h1>Título da Postagem</h1>
<ul>
<li>Autor: ...</li>
<li>Data: ...</li>
</ul>
</header>

<!-- Conteúdo/Corpo da Postagem -->
<article>
...
</article>

<!-- Rodapé da Postagem (comentários, por exemplo) -->
<footer>
...
</footer>
</section>
</main>

<!-- Sidebar -->
<aside role="complementary">
...
</aside>

<!-- Rodapé da Página -->
<footer role="contentinfo">
...
</footer>

No exemplo acima, podemos perceber que há mais de uma tag header, por exemplo. Falar em cabeçalhos pode ser subjetivo demais em uma página web. Temos, por exemplo, o cabeçalho da página, onde há o título e ocasionalmente um menu de navegação, e também temos cabeçalhos de postagens, cabeçalhos de widgets de uma barra lateral, entre outros exemplos. Para diferenciá-los, podemos utilizar os roles de landmark, fazendo assim a distinção entre diferentes tipos de cabeçalhos, como usado nesse exemplo.

Os seguintes valores para role são utilizados para pontos de referência, marcos de navegação dentro de uma página:

Role e Descrição Elemento relacionado
application: representa uma região da página que possui um software ou aplicação única, executando uma série de tarefas para o usuário. É um local onde tecnologias assistivas devem também retornar as teclas de navegação do navegador de volta para a aplicação web nessa região.

Nota: é extremamente recomendável utilizar este role com muita moderação.

 Não há um elemento específico relacionado, sendo recomendado utilizar uma tag div genérica.
banner: uma região que contém o cabeçalho principal de uma página ou título interno de uma página. A maior parte do conteúdo de um banner é comum em todas as páginas do website, ao invés de uma específica. Tipicamente inclui logos, navegações, o cabeçalho principal da página e mecanismo de pesquisa.

Nota: em qualquer documento ou página, o autor deve criar não mais que um elemento com este role.

 Não há um elemento específico relacionado, sendo recomendado utilizar uma tag header.
